1 Formulas

1.1 What are formulas?

A lot of material (more than) inspired from a DataCamp tutorial

?formula

It is a quoted, i.e. non evaluated expression:

x <- 1
x * 10
## [1] 10
quote(x * 10) 
## x * 10
typeof(quote(x * 10))
## [1] "language"
class(quote(x * 10))
## [1] "call"
eval(quote(x * 10))
## [1] 10

An example: Sepal.Width ~ Petal.Width + log(Petal.Length) + Species

1.2 Response(s) and predictor(s)

In the example above, we model the sepal width as a function of the petal width and length and the species. Formulas can be one or two sided:

One sided:

e <- ~ x + y + z

Two sided

f <- y ~ x + b

In a comparison

length(e)
## [1] 2
length(f)
## [1] 3
e[[1]]
## `~`
e[[2]]
## x + y + z
f[[3]]
## x + b

What is stored in a formula?

1.3 Writing formulas

1.3.1 In plain

y ~ x
## y ~ x
~ x + y + z
## ~x + y + z
g <- y ~ x + b

1.3.2 Programmatically

"y ~ x1 + x2"
## [1] "y ~ x1 + x2"
h <- as.formula("y ~ x1 + x2")
h <- formula("y ~ x1 + x2")
as.formula(paste(c("~ x","y","z"),collapse="+"))
## ~x + y + z

1.4 Formula operators

  • + to add terms
  • - to remove terms
  • * for crossing
  • : for interaction
  • %in% for nesting
  • ^ for limit crossing to the specified degree
  • | for conditionals (sometimes, not base R) ### + and - Combine predictors. Note that + implies that the predictors are independent i.e. in a linear model examples, they would be orthogonal
y ~ x1 + x2
## y ~ x1 + x2

Remove predictors

y ~ x1 - x2
## y ~ x1 - x2

1.4.1 *, :, %in% and ^

Important for regression, essential to describe interaction terms

Note that these are the same:

y ~ x1 * x2
## y ~ x1 * x2
y ~ x1 + x2 + x1:x2
## y ~ x1 + x2 + x1:x2

1.5 Caveats

Formulas usually just involve variables and factor names, they can also use arithmetic expressions. Imagine we want a model with x and x^2, writing the following fails:

terms(y ~ x + x^2)
## y ~ x + x^2
## attr(,"variables")
## list(y, x)
## attr(,"factors")
##   x
## y 0
## x 1
## attr(,"term.labels")
## [1] "x"
## attr(,"order")
## [1] 1
## attr(,"intercept")
## [1] 1
## attr(,"response")
## [1] 1
## attr(,".Environment")
## <environment: R_GlobalEnv>

Why? Because of the above where ^2 is crossing x to the second degree (and we have only one variable)

terms(y ~ x + x * x)
## y ~ x + x * x
## attr(,"variables")
## list(y, x)
## attr(,"factors")
##   x
## y 0
## x 1
## attr(,"term.labels")
## [1] "x"
## attr(,"order")
## [1] 1
## attr(,"intercept")
## [1] 1
## attr(,"response")
## [1] 1
## attr(,".Environment")
## <environment: R_GlobalEnv>

Rather we need to encapsulate arithmetic operations in I(), the as-is operator

terms(y ~ x + I(x^2))
## y ~ x + I(x^2)
## attr(,"variables")
## list(y, x, I(x^2))
## attr(,"factors")
##        x I(x^2)
## y      0      0
## x      1      0
## I(x^2) 0      1
## attr(,"term.labels")
## [1] "x"      "I(x^2)"
## attr(,"order")
## [1] 1 1
## attr(,"intercept")
## [1] 1
## attr(,"response")
## [1] 1
## attr(,".Environment")
## <environment: R_GlobalEnv>
terms(y ~ I(3 * x))
## y ~ I(3 * x)
## attr(,"variables")
## list(y, I(3 * x))
## attr(,"factors")

## y               0

1.7 Modifying formulas

For example, we could update our f formula from above

f
## y ~ x + b

Here, ~. is a special variable that represent the formula in f

update(f,~. -b)
## y ~ x

Arguably, one will most often simply rewrite the formula, using e.g. paste, but being able to dynamically update a model can come in handy

reformulate(termlabels=all.vars(e),intercept = FALSE,response="w")
## w ~ x + y + z - 1
